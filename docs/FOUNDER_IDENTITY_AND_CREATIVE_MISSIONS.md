# Founder Identity and Creative Missions

This document defines how the player-created Founder shapes the story of NEXUS Worlds.

## Core Rule

Making the player is a vital part of the story.

The player is not just selecting a skin. They are creating the Founder identity that shapes their own version of the world.

Every player should feel:

```text
This is my Founder.
This is my story.
This civilisation grows because of my choices.
```

## First-Person Founder POV

The player enters NEXUS Worlds in first-person.

The player should be able to customise their Founder however they want.

MVP customisation can be simple.

Later customisation should be deep.

## Character Creation Should Affect Story

Founder choices should influence:

- starting career/path
- first NPC trust bonuses
- first tools/resources
- starter mission flavour
- dialogue responses
- early civilisation strengths
- personal story tone

Example:

```text
Architect Founder: Mason trusts you faster and shelter progresses slightly quicker.
Medic Founder: Ava trusts you faster and colony health starts stronger.
Engineer Founder: Theo trusts you faster and research starts quicker.
```

## Career Paths

Starting paths:

- Architect
- Engineer
- Medic
- Farmer
- Scout
- Guardian

These should not lock the player out of content. They should shape the opening story and strengths.

## Creative Missions

The game needs creative missions and challenges so it feels worth playing.

Missions should not just be fetch quests.

They should make the player feel like they are shaping civilisation.

## First Storm Demo Missions

### Mission 1: Enter The Nexus

Goal:

- create Founder profile
- choose career
- load into Origin Planet

### Mission 2: Meet Mason

Goal:

- walk to Mason
- inspect his goal
- learn the shelter problem

### Mission 3: Give Tools

Goal:

- give basic tools to Mason
- unlock first shelter project

### Mission 4: The Storm Warning

Goal:

- trigger world event: A storm is coming
- watch agents react

### Mission 5: Secure The Camp

Goal:

- assign or influence agents
- protect food
- improve safety
- start shelter progress

### Mission 6: Advance Day

Goal:

- press Advance Day
- see what agents did
- read daily summary

### Mission 7: Build First Shelter

Goal:

- reach 100 percent shelter progress
- unlock Settlement Age requirement

### Mission 8: First Council

Goal:

- agents debate next priority
- player influences decision
- choose food, safety or research focus

## Challenge Types

NEXUS Worlds should include several challenge types:

### Survival Challenges

Examples:

- storm incoming
- low food
- damaged shelter
- sickness risk
- resource shortage

### Building Challenges

Examples:

- build first shelter
- repair camp after storm
- build farm
- create workshop
- upgrade storage

### Research Challenges

Examples:

- invent better tools
- discover construction
- unlock farming
- start electricity later

### Social Challenges

Examples:

- Mason and Elsie disagree about priorities
- Rex wants patrols while Theo wants research
- Ava warns the colony is exhausted
- player must guide the council

### Exploration Challenges

Examples:

- Nova finds new land
- scout safer ground
- discover stone deposit
- reveal water source later

### Founder Trials

Long-term original quest chain:

- Survive The First Storm
- Build The First Settlement
- Discover Power
- Create The First AI Lab
- Unlock Anti-Gravity
- Build The Spaceport
- Open The Central Hub Portal

## Mission Design Rule

Each mission should include:

- clear objective
- reason it matters
- NPC reaction
- visible world change
- activity feed update
- reward or progress

Example:

```text
Objective: Give tools to Mason.
Reason: Shelter cannot start without tools.
NPC reaction: Mason thanks the Founder.
World change: shelter frame appears.
Feed update: Mason started the shelter project.
Reward: Construction progress unlocked.
```

## Story Should Be Personal

The story is not one fixed story.

It is the player's version of events.

Different Founder choices should create different early story flavours.

Example:

```text
Architect path = builder story.
Medic path = survival and care story.
Engineer path = technology story.
Scout path = discovery story.
Guardian path = protection story.
Farmer path = food and growth story.
```

## Acceptance Criteria

The system works when:

- player can create their Founder
- player can choose career/path
- career affects opening flavour or stats
- first missions guide the player naturally
- missions create visible world changes
- agents react to player decisions
- activity feed records the story
- player feels this is their own civilisation story

## Bottom Line

NEXUS Worlds should make character creation and missions part of the story.

The player makes the Founder, enters in first-person, shapes the world through creative challenges, and watches AI citizens build a civilisation from those choices.
