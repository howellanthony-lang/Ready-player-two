# Ready Player Two / NEXUS Worlds

Ready Player Two is the repository name and creator-facing project. NEXUS Worlds is the original public-facing world brand for the playable prototype.

The game is a visual AI simulation where characters live inside a small world, talk to each other, remember events, form relationships, pursue goals, and react to player-created world events.

## Core Vision

The player watches a living world evolve rather than controlling every character directly.

Players should be able to:

- Watch AI agents move, talk, and react.
- Inspect character goals, memories, moods, and relationships.
- Trigger world events such as storms, discoveries, shortages, or celebrations.
- Create custom characters and worlds later.
- Save and replay emergent stories later.
- Share worlds, story logs, and templates later.

## Game Feel

The tone should be cosy, readable, lightly chaotic, and story-driven. The MVP should feel like a toy world that is easy to understand visually.

Useful inspiration:

- The Sims-style people watching.
- Animal Crossing-style cosy character presence.
- Dwarf Fortress and RimWorld-style emergent stories.
- AI Town-style autonomous agents.
- Original NEXUS Worlds civilisation evolution.

Do not copy protected brands, characters, locations, stories, logos, music, or exact visual designs from other IP.

## MVP Goal

Build a working browser prototype where:

- A visible 3D world loads.
- The player can move with WASD.
- Six starter citizens are visible.
- Clicking a citizen opens an inspector.
- An activity feed shows important events.
- A civilisation panel explains the current world state.
- A local world event input adds a feed entry.

Only after this works should the project add persistent Convex state, real AI conversations, multiplayer, VR, or advanced world generation.
