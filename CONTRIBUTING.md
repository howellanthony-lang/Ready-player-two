# Contributing To NEXUS Worlds

The current priority is:

```text
NEXUS Worlds: First Storm Demo
```

## Workflow

1. Pick or create an issue.
2. Create a feature branch.
3. Make focused changes.
4. Test locally.
5. Open a pull request.
6. Complete the PR checklist.

## Branch Names

Examples:

```text
feature/founder-controller
feature/mission-tracker
fix/mason-interaction
chore/update-docs
```

## Main Branch

Keep `main` stable.

Do not merge unfinished experiments into `main`.

Use `/research` for external references and experiments.

## Scope Rule

Do not expand beyond the First Storm Demo until it works well.

Delay larger systems like multiplayer, vehicles, large cities, space travel, large economy systems and complex AI autonomy.

## Asset Rule

Use Git LFS for Unreal binary assets.

Only add assets that the project is allowed to use.

## Documentation Rule

Update docs when behaviour changes.

Important docs:

- `docs/MASTER_UNREAL_GAME_MODEL.md`
- `docs/UNREAL_VERTICAL_SLICE_BUILD_PLAN.md`
- `docs/BUILD_INSTRUCTIONS.md`
- `docs/CODING_GUIDELINES.md`

## Quality Rule

A change is useful if it makes the First Storm Demo clearer, more playable, more alive, more emotionally engaging, or easier to maintain.
